The importance of software estimation is utmost, as it is one of the most crucial activities for software project management.
Although numerous software estimation techniques exist, the accuracy achieved by these techniques is questionable. This work
studies the existing software estimation techniques for Agile software development (ASD), identifies the gap, and proposes a
decentralized framework for estimation of ASD using machine-learning (ML) algorithms, which utilize the blockchain technology.
The estimation model uses nearest neighbors with four ML techniques for ASD. Using an available ASD dataset, after the
augmentation on the dataset, the proposed model emits results for the completion time prediction of software. Use of another
popular dataset for ASD predicts the software effort using the same proposed model. The crux of the proposed model is that it
simulates blockchain technology to predict the completion time and the effort of a software using ML algorithms. This type of
estimation model, using ML, making use of blockchain technology, does not exist in the literature, and this is the core novelty of
this proposed model. The final prediction of the software effort integrates another technique for improving the calculated
estimation, the standard deviation technique proposed by the authors previously. This model helped lessening the overall mean
magnitude of relative error (MMRE) of the original model from 6.82% to 1.73% for the augmented dataset of 126 projects. All four
ML techniques used for the proposed model give a better p-value than the original model using statistical testing through the
Wilcoxon test. The average of the MMRE for effort estimation of all four techniques is below 25% on a dataset of 136 projects. The
application of the standard deviation technique further helps in lessening the MMRE of the proposed model at 70%, 80%, and
90% confidence levels. The work will give insight to researchers and experts and open the doors for new research in this area.

software. As per McConnell [1], ABEE has three constraints for
achieving the best possible accuracy. First, the difference
between the base and the software under the estimation should

be minimal. The second constraint is that both the software
should use the same programing language, and the third constraint is that the type of the software under the estimation
should not differ from the type of the base software. Taking

For the prediction of software completion time, the proposed model utilizes the augmented version of the dataset for
projects from software houses in Pakistan. Three inputs of

cost help in making new predictions. Although the original
model [75] gave an estimated cost as an output, it is used as

the output of estimated time using four ML techniques: RF, LR,

neighbors; all ML algorithms use the nearest neighbors.
The hybrid approach initially using KNN and then the
predicting with LR, RF, SVR, and XGBoost ensures that the
model trains on the most relevant subset of data for each prediction. Instead of training models on the entire dataset, the
KNN approach selects local neighbors of similar projects,
which train the models for making more targeted predictions.
This allows accurate predictions, where the models learn from a
subset of the data that is more appropriate and related to the
input features. LR assumes a linear relationship between the
input features and the target variables; the core reason for using
this technique was on the grounds of its simplicity and its
ability to serve as a baseline model. In RF, as there are multiple
decision trees and it is an ensemble-based technique, it is very
effective in capturing nonlinear relationships between input
features and target variables. In software estimation, the relationship between size and effort or between size and time does
not follow the economies of scale [1]. The use of RF will help in
providing better prediction for such cases where linear relationships do not exist clearly, and it is also robust against overfitting
and also supports larger datasets. SVR utilizes the kernel trick
technique to map data into higher-dimensional spaces, which
allows SVR to capture complex patterns that linear models can
miss, but it can struggle with larger datasets. XGBoost, a gradient boosting ensemble method, further enhances predictive
power by iteratively optimizing decision trees to minimize
errors, effectively handling nonlinear relationships and scaling
efficiently with larger datasets. To accommodate for nonlinear
relationships handling, the model uses RF, SVR, and XGBoost
techniques; considering three techniques, SVR is ideal for smaller datasets, while RF and XGBoost can cater for larger datasets.
Another reason to predict using four different techniques is
also that the estimator will look for convergence among the
four techniques. If three or all four predictions are converging
with each other, this will give a stronger element of confidence
to the estimator about the prediction.

numpy.tile(), which was used for repeating the original dataset

empirically on the observed error distribution of the LR predictions. The slightly tighter bounds help control the standard
deviation, reduce overestimation bias, and keep the confidence
intervals closer to the base LR estimates. This approach balances the need for realistic variance representation while minimizing MMRE at 70%, 80%, and 90% confidence levels,
thereby improving the reliability of the refined estimate. The
