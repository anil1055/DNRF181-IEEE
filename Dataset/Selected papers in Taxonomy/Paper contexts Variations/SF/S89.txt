# **Applying Radial Basis Function Neural Networks Based on Fuzzy ** **Clustering to Estimate Web Applications Effort **

Abdelali Zakrani [1], Ali Idri [1]

**Abstract** – *An important research issue in software project management is how to predict*
*accurately an effort for the software project to develop at an early stage. Achieving high accuracy*
*when estimating the effort of the software under development would supply effectively the project*
*managers in leading to projects finished on time and within budget. Unfortunately, existing*
*software effort estimation techniques are still unable to provide acceptable estimates. This paper*
*investigates the application of Radial Basis Function Neural Networks (RBFN) based on fuzzy*
*clustering in web development effort estimation. The proposed model is designed by integrating*
*the principles of RBFN and the fuzzy C-means clustering algorithm. The architecture of the*
*network is suitably modified at the hidden layer to realize a novel neural implementation of the*
*fuzzy clustering algorithm. Fuzzy set-theoretic concepts are incorporated at the hidden layer,*
*enabling the model to handle uncertain and imprecise data, which can significantly improve the*
*accuracy of the obtained estimates. MMRE and Pred are used as measures of prediction accuracy*
*for the undertaken study. The results show that an RBFN using fuzzy C-means performs better*
*than an RBFN using hard K-means. This study uses data on web applications from the Tukutuku*
*database.* ***Copyright © 2010 Praise Worthy Prize S.r.l. - All rights reserved.***

**I.** **Introduction**

Accurate and reliable effort estimate of the software

project under development is the most crucial
information for the project managers. Having this vital
information allows them to make more effective project
planning, control and staffing, therefore leading to
project delivered on time and within budget. Whereas,


managing the software project without accurate estimate
makes the task of the project manager more difficult and
stressful. As a result, he/she may take wrong decisions
threatening the success of the whole project. As case in
point, when the software project is under-estimated, the
manager probably will under-staff it (resulting in staff
burnout), under-scope the quality assurance effort
(running the risk of delivering poor quality product), and
set too short schedule (resulting in loss of credibility as
the deadlines are missed).

Aiming at assisting project managers in their tasks,
numerous software development effort estimation
techniques have been developed over the last three
decades [1]. In a recent systematic review [2], Jorgensen
and Shepperd identified 304 papers published in 76
journals dealing with the development and the
improvement of software effort estimation approaches
(more details can be found in [2]). These latter are based
on different techniques varying from expert judgment [3]
and statistical analysis [4] to artificial intelligence tools

[5].
In recent years, Artificial Neural Networks (ANNs)
have been emerged as successful modeling techniques
when seeking an approximation of a nonlinear
relationship between the independent and dependent
variables. Previously, Hornik et al. [6] have proved that
ANN are capable to model any measurable function. In
the context of software effort estimation, artificial neural
networks have been increasingly employed to model the

complex relationship linking the software attributes and
software development effort [7]-[8]-[9]. The most used
architecture network in the aforementioned references is
the multilayer perceptron (MLP). Unlike these latter, this
paper uses Radial Basis Function (RBFN) neural
networks to address estimation effort issue.

The use of Radial Basis Function Networks to predict
software development effort requires the determination
of some parameters. In an earlier work [10], we have
empirically evaluated the use of some crisp clustering
techniques in designing the middle layer of RBFN for
software effort estimation. Particularly, the two used
crisp clustering techniques were the K-means and the
APC-III algorithms. The two studies were based on the
COCOMO’81 and a web hypermedia dataset that
contains 53 software projects. We have found that the
RBFN designed with the K-means algorithm performs
better, in terms of effort estimates accuracy, than the
RBFN designed with APC-III algorithm for the two
datasets.
The purpose of the present paper is to assess and to
discuss the use of fuzzy clustering techniques, especially
the fuzzy C-means algorithm in designing RBFN
networks for software effort estimation. Instead of crisp
(no fuzzy) clustering techniques where each software
project belongs to exactly one cluster, fuzzy clustering
techniques will allow to each software project to belong
to more than one cluster with different degrees. Hence,
the designed model may handle correctly the imprecision
and the uncertainty when assigning a software project to
clusters. This study uses 53 web projects from Tukutuku
database [11].

The rest of this paper is structured as follows: Section
2 provides some background information about the effort
estimation literature. It is followed in Section 3 by a brief
description of RBF Networks. In Section 4, we describe
how fuzzy logic is incorporated in the hidden layer of the
RBFN networks by using fuzzy C-means. A description
of historical projects dataset and evaluation criteria is
presented in Section 5. Experiment design and results
obtained are addressed in Section 6 and 7 respectively. In
Section 8, the concluding remarks are stated and the
future work is discussed.

layer in which each neuron computes its output using a
radial basis function, which is in general a Gaussian
function, and an output layer which builds a linear
weighted sum of hidden neuron outputs and supplies the
response of the network (effort). A RBF neural network
configured for software effort estimation has only one

*V.1.* *Data Description*

The data used in the present study come from
collection of historical web applications project namely
Tukutuku dataset.

The Tukutuku dataset contains 53 Web projects [11].
Each Web application is described using 9 numerical
attributes such as: the number of html or shtml files used,
the number of media files and team experience (see
Table 1). However, each project volunteered to the
Tukutuku database was initially characterized using more
than 9 software attributes, but some of them were
grouped together. For example, we grouped together the
following three attributes: number of new Web pages
developed by the team, number of Web pages provided
by the customer and the number of Web pages developed
by a third party (outsourced) in one attribute reflecting
the total number of Web pages in the application
(Webpages). By analyzing statistically Tukutuku dataset,
we noticed that the data is skewed and containing several
outlier projects (Table 2).

**VII.** **Results of the Experimental Studies**

The following section presents and discusses the
results obtained when applying the RBFN to the web
dataset. The calculations were made using two software
prototypes developed with C language under a Microsoft
Windows PC environment. The first software prototype
implements the hard and the fuzzy C-means clustering
algorithms, providing both the clusters and their centers
from the web applications dataset. The second software
prototype implements a effort estimation model based on
an RBFN architecture in which the middle-layer
parameters are determined by means of the first software
prototype.

We have conducted several experiments with both the
hard K-means and the fuzzy C-means algorithms to
decide on the number of hidden units. These experiments
use the full web applications dataset for training.
Choosing the ‘best’ classification to determine the
number of hidden neurons and their centers is a complex
task. For software effort estimation, we suggest that the
‘best’ classification is the one that satisfies the following
two criteria:

 - It provides coherent clusters, i.e. the software
projects of a given cluster have satisfactory
degrees of similarity;

 - It improves the accuracy of estimates, in terms of
MMRE and Pred(0.25), produced by the RBFN .

To measure the coherence of clusters in the case of

fuzzy C-means algorithm, we use the well known fuzzy
clustering validity function developed by Xie and Beni

[36] and defined as follow:

RBFN, we need to optimize the widths of the Gaussian
kernels. For this issue we have conducted several

sensitive to individual predictions with excessively large

**VIII.** **Conclusion and Future Work**

In this paper, we have empirically studied the use of
two clustering techniques when designing RBF neural
network for software effort estimation. The two used
clustering algorithms are the well-known hard and the
fuzzy C-means. This study is based on one dataset
containing 53 web applications. We used the entire
dataset to train and test the designed RBFN. We have
found that the RBFN designed with the fuzzy C-means
algorithm performs better, in terms of the effort estimates
accuracy, than the RBFN designed with the hard Kmeans algorithm. To confirm this result, we are looking
currently in applying cross validation methods to
compare the generalization ability of hard and fuzzy Cmeans based RBFN for software development effort
estimation.
